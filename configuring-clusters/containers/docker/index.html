Docker

Dockerâ€‹

This section provides a very quick introduction for using dockers. For a more thorough documentation on Docker refer to:

  • https://docs.docker.com/get-started/
  • https://docs.docker.com/

What is Docker?â€‹

A Docker container is a stand-alone package of a piece of software that includes everything needed for running an application. Using Docker enables running your application on any environment and ensures that your code runs the same regardless of the environment.

Running an application via Dockerâ€‹

To run a command in a docker container, you can use the docker run command. For example, to execute the python3 program hello.py in the docker container parallelworks/python-tools:v3.6-pandas, instead of running the command

python3 hello.py

directly via the command line, you would run the above command in a docker container based on the docker image parallelworks/python-tools:v3.6-pandas:

docker run --rm  -i -v `pwd`:`pwd` -w `pwd` parallelworks/python-tools:v3.6-pandas python3 hello.py

The options used above for running the command docker run are :

  • --rm : automatically removes the container after exiting from the docker container (does not remove the files generated under the mapped volume with option -v)
  • -i : for running interactively
  • -v `pwd`:`pwd` : mounts* the current directory inside the docker container when the docker starts
  • -w `pwd` : sets the working directory inside the container to the current directory

For development and testing, you can also run commands interactively (e.g., bash or python) via a docker container and run several commands from within that container. For example to start a bash shell inside a docker container based on the image parallelworks/python-tools:v3.6-pandas run:

docker run --rm -t  -i -v `pwd`:`pwd` -w `pwd` parallelworks/python-tools:v3.6-pandas python

Running this command will open the interactive python session inside the python-tools docker container:

Python 3.6.4 (default, Feb 17 2018, 09:32:33)
[GCC 4.9.2] on linux
Type "help", "copyright", "credits" or "license" for more information.
>>>

Note that the option -t is required to run commands interactively in a docker container. To exit from the container, run:

exit

Building a Docker imageâ€‹

There are multiple ways of making a new docker image (detailed steps for each approach are below):

  1. Start by running a docker container (with docker run) based on an existing image with bash (e.g., from python:3.6). Install software and make necessary changes inside the container, and then exit from the docker and commit the docker as a new image.
  2. Build a docker from a Dockerfile. In this method, the commands for installing software and making changes are put in a Dockerfile. The docker image is then built via the docker build command (see Getting started from Docker Documentation: https://docs.docker.com/get-started/part2/#dockerfile).

In general, the first method is preferable for debugging and testing, but this method is not easy to replicate because the steps for making a docker image are not easily retrievable. Also, later on, it would be difficult to find out the installed software and configuration of docker images that are built this way. So, you could start building a docker via the first method and after installing, configuring and testing your software, you can create a new Dockerfile and add the required steps to it (the bash command history provides an easy method for retrieving the commands that were executed for installing software in the docker container).

Starting a docker from a containerâ€‹

Here are the steps for building

  • First, change to the directory that contains the necessary files for installing your software (e.g. software binary files).

  • Then start a container based on an existing image, e.g., python:3.6 :

    ~/user/dockers/python-pandas $ docker run -it -v `pwd`:`pwd` -w `pwd` python:3.6 /bin/bash

    Note that you should not use the option --rm to be able to save the container as an image at the end.

  • Install and setup your software, from the interactive bash command inside the docker container. For example, to install Pandas, run

    root@98a7c60197d3:/home/user/dockers/python-pandas#  pip install pandas

  • To exit from the container run the command exit:

    root@98a7c60197d3:/home/user/dockers/python-pandas#  exit

  • Now, you can make and image from your container:

    • First, find the container id by running

      docker ps -a

      to show all the containers. For example:

      CONTAINER ID  IMAGE       COMMAND      CREATED             STATUS                     PORTS  NAMES
      98a7c60197d3  python:3.6  "/bin/bash"  About a minute ago  Exited (0) 15 seconds ago         affectionate_brattain

    • Use the container ID, to make a new image from your container:

      docker commit 98a7c60197d3 python3-pandas

Building the docker from a Dockerfileâ€‹

  • Building the docker using a Dockerfile requires downloading the necessary files for installation (if any) and the Dockerfile in a directory.

  • This is an example of a Dockerfile based on a python3 docker container with Pandas and matplotlib:

    # Use python:3.6 as a base image
    FROM python:3.6

    # Install pandas and xlrd

    RUN pip install pandas  && \
        pip install matplotlib  && \
        pip install xlrd

    RUN useradd -ms /bin/bash  pyuser

    CMD ["python3"]

    More examples of Dockerfiles can be found in https://github.com/parallelworks/Dockerfiles

  • Build the docker image via the docker build command (from inside the folder that contains the Dockerfile):

    docker build -t python-pandas .

Sharing/publishing your docker imageâ€‹

To share your image on docker hub, follow the directions in https://docs.docker.com/get-started/part2/#share-your-image

Footnotesâ€‹

  1. Mounting a directory means making that particular directory accessible at a certain location inside the docker container. Only the mounted directories (and their sub-directories) are accessible via the docker container.
