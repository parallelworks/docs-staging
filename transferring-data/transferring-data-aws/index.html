Transferring Data to/from AWS S3 Storage

This page goes over everything you’ll need to transfer data to and from AWS storage, including how to set up an S3 bucket, create an IAM account, and add account permissions from your organization’s root user AWS account.

If you or your organization’s administrator have already completed these steps, you can skip to Transferring Your Data below.

Setting Up Your AWS Account​

Log In to AWS​

The process of logging in to AWS may look different for each user, depending on your organization and your account’s access permissions. If you log in at https://aws.amazon.com/, you’ll need to choose which type of account you have before entering your credentials. If you use AWS SSO (Single Sign-On) from Gmail's app list, you’ll automatically be directed to the account landing page.

After logging in, select Management console for the account you’ll be using.

In the following sections, the settings we use are recommended for high security and ease of use for non-root AWS users. Please feel free to adjust these settings to fit your organization’s needs.

Create a Bucket​

Navigate to the Amazon S3 page, which is in Services > Storage > S3. You can also enter “S3” in the search bar.

Click Buckets in the sidebar, then click Create bucket.

Enter a name for your bucket. Please note that the name you use cannot have spaces and must be unique from all other AWS bucket names.

Under Object Ownership, select ACLs disabled.

Under Block Public Access settings for this bucket, select Block all public access.

Under Bucket Versioning, select Disable. Optionally, add any tags that would be useful for you or your organization in the Tags box.

Under Default encryption > Encryption key type, select Amazon S3-managed keys. Under Default encryption > Bucket Key, select Enable.

Click Create bucket. On the next page, you’ll see the automated message Successfully created bucket “BucketName.”

Create a Policy​

Navigate to the Identity and Access Management (IAM) page, which is under Services > Security, Identity, & Compliance > IAM. You can also ****enter “IAM” in the search bar.

Navigate to the Policies page. You can select Access Management > Policies or if you’re on the IAM dashboard, click the number under Policies.

Click Create policy.

Click Service to expand the list of options. Select S3.

After you select a service, Actions will automatically expand. Under Access level, click List to expand it. Select ListBucket.

Expand Read. Select GetObject.

Expand Write. Select DeleteObject and PutObject.

Expand Resources. Next to bucket, click Add ARN.

A pop-up window will appear. Enter the name of the bucket you created earlier. Alternatively, you can open a new browser tab and copy the ARN from the bucket’s Properties section (Amazon S3 page > Buckets > BucketName > Properties).

Click Add.

Next to object, select Any. Click Next: Tags.

Click Add tag to include any tags that would be useful for you or your organization. When you’re done, click Next: Review.

Name your policy. Please note that the name you use cannot have spaces. Optionally, enter a description for your policy. Click Create policy.

On the next page, you’ll see the automated message The policy PolicyName has been created.

Create a User Group​

From the IAM page, navigate to the User groups page. You can select Access Management > User groups or if you’re on the IAM dashboard, click the number under User groups.

Click Create group.

Enter a name for your group. Please note that the name you use cannot have spaces.

In Attach permissions policies, select the policy you created. Click Create group.

On the next page, you’ll see the automated message GroupName user group created. **

Create an IAM Account​

From the IAM page, navigate to the Users page. You can select Access Management > Users or if you’re on the IAM dashboard, click the number under Users.

Click Add users.

Enter a name for your user. Please note that the name you use cannot have spaces. Do not enable console access. Click Next.

In User groups, select the group you created. Click Next.

Optionally, add any tags that would be useful for you or your organization in the Tags box. Click Create user.

On the next page, you’ll see the automated message User created successfully.

Create AWS Keys​

This section walks through the creation of an AWS access key, which functions similarly to SSH keys. For security, we recommend using AWS Security Token Service (STS) to create temporary keys for users. See About Data Storage and Transfer for more information.

From the Users page, click the name of the user you created. On the next page, click Security credentials.

Scroll down to Access keys and click Create access key.

Select Application running outside AWS. A message with best practices will appear at the bottom of the page. Click Next.

Optionally, add any tags that would be useful for you or your organization. We added “DeleteMe” for our key because it was meant as a temporary test key.

Click Create access key.

On the next page, you’ll see the automated message Access key created.

This is the only time you’ll have access to your secret access key. Click Download .csv file to save both keys to your device. We recommend treating your access keys with the same amount of caution that you apply to SSH keys.

Upload a Test File​

Navigate to the Amazon S3 page (Services > Storage > S3). Click the name of the bucket you created earlier. Click Upload.

Add your file to the bucket by dragging and dropping or clicking Add files or Add folder. When you’re done, click Upload.

On the next page, you’ll see the automated message Upload succeeded.

Working With AWS CLI​

To transfer data to/from S3 bucket storage, you’ll use the AWS Command Line Interface (CLI).

Because the AWS CLI is pre-installed on AWS clusters in the PW platform, you can enter commands directly into the IDE after logging in to the controller of an active AWS cluster. If you’re transferring data between S3 buckets and a folder, an on-premise cluster, or a remote device, you’ll likely need to install the AWS CLI first.

Check for AWS CLI​

Open a terminal or command line and navigate to your data’s destination. Enter which aws.

If the AWS CLI is installed, you’ll see a message that shows its location, such as /usr/local/bin/aws.

If the AWS CLI is not installed, you’ll see a message such as /usr/bin/which: no aws or aws not found.

Install AWS CLI​

Open a terminal or command line and navigate to your data’s destination.

Enter git clone [https://github.com/parallelworks/cloud-data-transfer](https://github.com/parallelworks/cloud-data-transfer) to clone PW’s GitHub repository to your device.

If the cloning is successful, you’ll see this message:

Navigate to the folder you just cloned with cd cloud-data-transfer.

Transferring Your Data​

Validate Your AWS Credentials​

Enter export AWS_ACCESS_KEY_ID="_____" with your AWS access key ID in the blank space.

Enter export AWS_SECRET_ACCESS_KEY="_____" with your AWS access key ID in the blank space.

Please be sure to include the quotation marks on both ends of your key IDs. There are characters inside AWS key IDs that systems will try to read as commands without the quotation marks.

Check Your AWS Credentials​

Enter env | grep -i aws to check that the system registered your input. If your entry was successful, you’ll see this message: 

AWS_ACCESS_KEY_ID=XX/xX0XxX00XXX
AWS_SECRET_ACCESS_KEY=XxX00XXxxXX0XX00XXXx000XxXXXx

If your entry was not successful, you won’t see a message:

demo@pw-user-demo:~/pw/cloud-data-transfer$ env | grep -i aws
demo@pw-user-demo:~/pw/cloud-data-transfer$

Check for S3 Files​

Enter aws s3 ls s3://bucket_name to display the files in your bucket. For this guide, we used a small text file named test.txt, so our command returned this message: 

demo@pw-user-demo:~/pw/cloud-data-transfer$ aws s3 ls s3://cloud-data-test
2023-02-08 16:37:56         28 test.txt

Transfer a File From S3​

Enter aws s3 cp s3://bucket_name/file_name ./ to download a file from S3 storage to your current directory. You’ll see this message:

demo@pw-user-demo:~/pw/cloud-data-transfer$ aws s3 cp s3://cloud-data-test/test.txt ./
download: s3://cloud-data-test/test.txt to ./test.txt

To download a file from S3 storage to a specific directory, enter its name after ./ with the aws s3 cp command. We used the storage directory in this example:

demo@pw-user-demo:~/pw/cloud-data-transfer$ aws s3 cp s3://cloud-data-test/test.txt ./storage
download: s3://cloud-data-test/test.txt to ./storage

Transfer a File to S3​

Enter aws s3 cp file_name s3://bucket_name/ to transfer a file to your S3 bucket.

For this guide, we first used the command mv to rename our test file test.txt to test_upload.text, then checked our directory with ls: 

demo@pw-user-demo:~/pw/cloud-data-transfer$ mv test.txt test_upload.txt
demo@pw-user-demo:~/pw/cloud-data-transfer$ ls
install_aws_cli.sh  LICENSE  README.md  run_and_process_aws_test.sh  storage  test_aws_cli.sh  test_upload.txt

Then we uploaded our test file to S3 storage: 

demo@pw-user-demo:~/pw/cloud-data-transfer$ aws s3 cp test_upload.txt s3://cloud-data-test/
upload: ./test_upload.txt to s3://cloud-data-test/test_upload.txt

You can check your S3 files again with aws s3 ls s3://bucket_name:

demo@pw-user-demo:~/pw/cloud-data-transfer$ aws s3 ls s3://cloud-data-test/
2023-02-08 16:37:56         28 test.txt
2023-02-08 17:19:24         28 test_upload.txt

You can also check that your file was uploaded by navigating to your bucket in the AWS management console.

Delete a File From S3​

Enter aws s3 rm s3://bucket_name/file_name to delete a file. You’ll see this message: 

demo@pw-user-demo:~/pw/cloud-data-transfer$ aws s3 rm s3://cloud-data-test/test_upload.txt
delete: s3://cloud-data-test/test_upload.txt

You can check your S3 files again with aws s3 ls s3://bucket_name:

demo@pw-user-demo:~/pw/cloud-data-transfer$ aws s3 ls s3://cloud-data-test/
2023-02-08 16:37:56         28 test.txt

You can also check that your file was uploaded by navigating to your bucket in the AWS management console.
