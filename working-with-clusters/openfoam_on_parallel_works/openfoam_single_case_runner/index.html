OpenFOAM Single Case Runner

The OpenFOAM Case Runner workflow is a simple way to run your large OpenFOAM studies in the cloud.\ Launch your study in just a few easy steps, detailed below.

Configure Resources​

Configure your resources by either editing or adding a new resource to meet the computing requirements of your OpenFOAM study.Edit your number of desired workers per instance.\ Tip: For optimal performance, set workers/instance to 1. This will use one worker to run your study across all available CPUs on your selected instance.

After editing your resources, power “on” your desired resource and select the maximum number of active workers you would like to run your study on. Note: The number of workers will increase in increments corresponding to the number of CPUs on your selected instance.

Create a New Workspace​

Create a new workspace to manage your working files for the study. Select “View All Workspaces” and in the All Workspaces page, select “Create New” in the top right corner of the screen. Name your workspace and then select “Done.”

Upload Input Files​

The Case Runner workflow requires 2 standard input files:\ (1) A tar file of the standard case input files (system, constant, 0.org etc.)\ (2) An OpenFOAM decomposeParDict file to guide the decomposition of your mesh\ Upload these files to the platform by selecting Upload Data or clicking “load your own data” in the empty Workspace box.

Configure Workflow​

Customize the exact commands that you would like OpenFOAM to run by editing the runOpenfoam.sh script in the Workflow/Single_Case_Runner editing interface\

In the Workflows Interface, select the Single Case Runner workflow.

Select the "Run Files" tab in the edit interface to access the IDE and make your edits. The original template script is displayed here:

You can edit the commands to customize the workflow to run your job. A simple example can be seen below. The platform is flexible and can accomodate any preferred OpenFOAM run codes.

Once you have completed any edits, save your workflow and return to the COMPUTE screen.

Launch Workflow​

Select the OpenFOAM Runner tile to access the workflow execution interface\

Using the drop downs in the workflow execution interface, select the input files for each required input.\ Indicate the number of subdomains desired, which should match the numberOfSubdomains value in your decomposeParDict configuration file, as well as the product of the n values under the heirarchical configuration highlighted below.

Once all your input configurations are set, select “Execute” to launch your workflow.

Monitor your Study​

Upon execution of the run, a workflow monitor will appear to display the status of your job. There are several job monitors, detailed below.

Progress Track progress of your study at the job execution level.

Data Browser See status for each job stage in your workflow.

Run Tail See standard run tail output for your study.

Run Directory View std.out and other execution-related output files.

Access these run details at any time during the execution of your job by clicking the Run Monitor above the Resource Monitor on the COMPUTE screen.

Visualize Results​

Upon completion of the run, your results will display as complete (in green) in the workspace under which it was launched.\ The OpenFOAM case runner will result in a packaged tar file complete with all results files. Select the eye icon to download the file.\ \ Visualize results as you traditionally view results of your OpenFOAM studies.
