Setup the Parallel Works Cloud Connector in HELYX-OS

Once the geometry has been loaded into HELYX-OS and the mesh properties are all defined, click Run Cloud button as shown here

A new window will appear as showed in the image below.

By default, the Execution Driver option is set to Localhost which means that all OpenFOAM runs will be executed on the local machine where HELYX-OS is installed. In order to enable the runs in the Parallel Works Cloud, change Execution Driver to Parallel Works and the Run Cloud panel will look like in the following image.

The first thing to do in order to setup the Parallel Works Cloud Connector is to navigate to the Parallel Works website, create an account and generate an API to connect the Parallel Works platform to HELYX-OS.\ Click the ‚ÄúGet a Parallel Works API key‚Äù as shown in the image above to be directed to a custom Parallel Works sign up page in your browser. Our guess is, if you're reading this, you've already gotten to this step!

Locate and Copy API Key‚Äã

In the Parallel Works platform, navigate to the Account tab to generate an API key. Copy your personalized API key and paste your it in the designed text field in the Run Cloud Window.\ Make sure "helyxos_runner" is set in Run Workflow and "helyxos_workspace" is specified in the Run Workspace text field. Be sure to protect your API key. This is as important as your Parallel Works account password.

Helyx-OS Run Cloud Window‚Äã

Now Helyx-OS is cloud enabled. The Pull Results from Cloud option, which is enabled by default, allows the user to download the case data (e.g. mesh files, solution results, etc.) from the remote execution machine in the Parallel Works Cloud to the case directory in the local machine where HELYX-OS is installed.\ \ Selecting the Download Last Cloud Results button allows the user to download the case data from the latest run performed on the Parallel Works cloud.
