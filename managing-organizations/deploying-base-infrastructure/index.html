Deploying Base Infrastructure

This page details how to set up access in AWS, Google, and Azure so that the PW platform has access to manage infrastructure, clusters, billing, storage, and usage data.

Note

The steps included on this page should be completed by a cloud engineer in your organization.

Setting Up AWS Credentials​

To get started quickly, you can create a new IAM account and assign the AdministratorAccess role to it.

Alternatively, you can create policies listed in AWS Policies below, attach them to a new role, then attach that role to an IAM user or user group.

Create an IAM Account​

You can create an IAM account from the management console, the CLI, or the API. Instructions for each method can be found in the AWS documentation here.

Create a User Group​

If you want a policy to apply to several users instead of just one, you can create a user group by following the AWS documentation here.

Create a Role​

You can create roles from the management console, the CLI, or the API. Instructions for each method can be found in the AWS documentation here.

Create a Policy​

You can create policies from the management console, the CLI, or the API. Instructions for each method can be found in the AWS documentation here.

Create AWS Keys​

AWS uses access keys that function similarly to SSH keys, with both a private and a public version that you’ll see when they’re created.

You can create AWS access keys for IAM users by following the AWS documentation here.

Alternatively, you can use the AWS Secret Token Service (STS) to request temporary, limited credentials. For more information about STS, see the AWS documentation here.

For more information about AWS keys and security best practices, see this FAQ on AWS’s site.
